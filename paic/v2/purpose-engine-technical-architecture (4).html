Core AI Definitions

The "AI Magic" Explained

The Purpose Engine uses four key AI techniques to personalize education while protecting student privacy. Each technique serves a specific function in the system architecture.

AI Technique #1

Joint Embedding

A neural network architecture that maps disparate data types (text, behavior patterns, assessments) into a shared mathematical space where semantic relationships can be computed.
üîç Where It's Used
"The Purpose Engine uses Joint Embedding to discover that Maya's love of Minecraft world-building shares the same spatial reasoning patterns as architectural design‚Äîconnecting her hobby to her career interest."
AI Technique #2

Semantic Alignment

A constraint-satisfaction process that ensures generated content maintains fidelity to source requirements while transforming surface-level presentation.
üîç Where It's Used
"When the Career Bridge transforms a stoichiometry worksheet into an 'Eco-Material Design Challenge,' Semantic Alignment verifies that 100% of TEKS standard C.8(C) objectives are preserved."
AI Technique #3

Edge-Based Inference

Model inference executed locally on school infrastructure rather than cloud servers. Student data never leaves the campus network boundary.
üîç Where It's Used
"All behavioral pattern detection runs via Edge-Based Inference on the school's local server. Raw student data is processed and discarded; only derived insights are stored."
AI Technique #4

Cross-Modal Reasoning

The ability to synthesize understanding across different input modalities (text, audio, visual) to infer context that no single modality reveals.
üîç Where It's Used
"The Future Self Mentor uses Cross-Modal Reasoning to detect that a student's video reflection shows low energy despite positive words, adjusting its response to be more supportive."

Why On-Premise Matters
By running inference locally, we eliminate the need to transmit student data to external cloud services. This "Edge-Based Inference" model means the school retains full custody of all data.

Data Flow & Processing

Privacy Pipeline

All data follows a strict pipeline that separates raw input from stored insights. Raw data is processed and immediately discarded; only validated, anonymized insights persist.

1
Capture
Teacher records voice note, uploads photo, or enters text observation
2
Process
Edge AI extracts semantic insights locally; raw media deleted within 60 seconds
3
Validate
Human-in-the-loop: Teacher or counselor confirms AI observation before storage
4
Store
Only validated insight text stored to Purpose Profile; full audit trail maintained

Privacy by Design

Core Principles

üóëÔ∏è Data Minimization

Raw media (voice, photos, video) is processed and deleted within 60 seconds. Only derived semantic insights are retained. We never store what we don't need.

üë§ Human-in-the-Loop

AI observations are "soft memory" until validated by a human educator. This prevents algorithmic bias from becoming permanent student record data.

üè´ Edge Processing

All AI inference runs on local school infrastructure. Student data never traverses the public internet or reaches third-party cloud servers.

üìã Role-Based Access

BYOD users see only student names and goals. Full profiles require elevated permissions tied to educational role (teacher of record, assigned counselor).
